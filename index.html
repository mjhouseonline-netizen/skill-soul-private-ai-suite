Recommended delivery

Use this page as the front door, not the installer folder.

Customers should land on a polished page, click one download button, and install. They should never have to browse your build folders or open developer files.

  • Host the landing page on Netlify and keep the installer in a GitHub Release.
  • Use the GitHub Release page as the primary installer destination for the `.exe`.
  • Keep the installer filename stable, even if the product branding on the page evolves.
  • Use your Skool community as the support and onboarding destination.

FAQ

Answer the questions before they become support tickets.

Should I attach the installer to an HTML file?

No. Use the HTML page as the presentation layer and the installer as a direct download file. Customers should click a button and download the `.exe` cleanly.

How should the large installer be delivered?

Keep the landing page on Netlify and upload the installer to a GitHub Release. The page can then send customers straight to the latest release download without bundling the `.exe` into the website deploy.

What is the best public delivery flow?

A simple branded Netlify page, a GitHub Release for the Windows installer, and clear support links to Skill & Soul Studio and your Skool community. That keeps the experience clean and trustworthy.
